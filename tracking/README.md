# tracking/ — session plans, progress tracker, decision log

Per `docs/STRUCTURE.md` §2, §7. Holds:

- **`plan-sessions` output** — the structured multi-session implementation plan
  (created when the `plan-sessions` skill runs against the W0–W10 work
  breakdown in `docs/STRUCTURE.md` §7).
- **Progress tracker** — session-by-session status.
- **`DECISIONS.md`** — the Decision & Change Log (seeded from `docs/STRUCTURE.md`
  §10 locked decisions; record new decisions here as continued planning
  proceeds).
