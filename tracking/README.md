# tracking/ — session plans, progress tracker, decision log

Per `docs/STRUCTURE.md` §2, §7. Holds:

- **`IMPLEMENTATION_PLAN.md`** — the structured multi-session implementation plan
  (`plan-sessions` output against the W0–W10 work breakdown in
  `docs/STRUCTURE.md` §7). 12 dependency-ordered sessions; the source of truth
  for what each implementation session does and its completion criteria.
- **Progress tracker** — session-by-session status.
- **`DECISIONS.md`** — the Decision & Change Log (seeded from `docs/STRUCTURE.md`
  §10 locked decisions; record new decisions here as continued planning
  proceeds).
